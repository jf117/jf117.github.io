WampServer服务器配置

发表于 2017-01-04 | 分类于 服务器

一、下载wampserver，正常安装（如：安装在D盘下）

二、You don’t have permission to access / on this server. 没权限

解决方案:

  1. 打开D:\wamp\bin\apache\Apache2.2.21\conf\httpd.conf
  2. 按ctrl+f 搜索documentRoot 大概188行左右的位置 directory标签添加 Allow from all
  3. 大概232行 Allow from 127.0.0.1 改成all
  4. 重启服务器

三、如何配置网站根目录

解决方案:

  1. 打开D:\wamp\bin\apache\Apache2.2.21\conf\httpd.conf
  2. 找到178行DocumentRoot “D:/wamp/www/“更改路径
  3. 找到206行更改路径
  4. 保存，重启服务器

四、如何配置虚拟主机

解决方案：

  1. 打开D:\wamp\bin\apache\Apache2.2.21\conf\httpd.conf
  2. 搜索Virtual hosts 找到第467行 会看到一个路径
  3. 根据路径找到一个文件打开conf/extra/httpd-vhosts.conf
  4. 更改DocumentRoot为网站的根目录
  5. 更改ServerName baidu.com
  6. 打开hosts文件 C:\Windows\System32\drivers\etc
  7. 写成 127.0.0.1 www.study.com 127.0.0.1 study.com
  8. 保存
  9. 重新启动服务器

五、配置反向代理

  1. 打开D:\wamp\bin\apache\Apache2.2.21\conf\httpd.conf文件
  2. 打开其中的proxy_module和proxy_http_module服务
  3. 打开conf/extra/httpd-vhosts.conf文件
  4. 在指定的根目录下添加ProxyRequests Off和ProxyPass /api http://api.botue.com（其中http://api.botue.com为反向代理服务器）

前端模块化

发表于 2017-01-04 | 分类于 javaScript

一、为什么要前端模块化

  • 公共功能进行封装实现复用
  • 灵活解决依赖关系
  • 解决全局变量污染

二、如何实现前端模块化

  • js在浏览器端本身是不具备模块化的技术的，需要自己进行封装来实现模块定义以及加载，其实现遵照一定的规范（COMMOMJS）
  • nodejs是运行在服务端的js，它是按着COMMONJS的规范实现的模块化
  • 经实践发现，浏览器端js按着COMMONJS的规范实现模块化，有很多不足，此时，就有人在COMMONJS基础上重新定义了规范AMD（async module definition）为浏览器端定义的规范，代表：require.js
  • 随着前端开发的演变，又有人定义了另一个浏览器端模块化的规范CMD（common module definition）代表：seajs

三、前端模块化使用requirejs

requirejs：

  • 本身就是几个js文件，它按照AMD规范来实现模块的定义和加载规则

1.定义模块

通过define()方法定义一个模块

  • 基本使用

  • 依赖注入，[]内是依赖的模块，function里的形参是注入

  • 返回值
    定义模块时可以有返回值，但不是必须的，返回值可以是任何类型的，但是返回对象更有意义

2.加载模块

通过require()方法加载模块

  • 基本使用

  • 依赖注入

3.入口文件

通过为引入require的标签加入data-main属性，其属性值为某一个模块的路径，此模块可以自动被加载并执行

4.加载路径

requirejs加载模块时遵照一定的规则，可以分成以下几种情况

  • 当没有入口文件时，加载路径以引入requirejs的页面为准
  • 当有入口文件时，加载路径以入口文件为准
  • 通过配置可以自定义路径

5.配置项

  • 配置基础路径

  • 配置路径

  • 配置不支持模块化的插件

  • 默认有很多第三方插件已经按照模块化的形式进行开发了

  • 要将不支持模块化的插件当成模块来使用，那就要指定依赖和返回值，通过deps和exports

四、匿名模块和具名模块

模块名必须和config的paths里的别名一致
