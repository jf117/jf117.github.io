前端模块化

前端模块化

为什么要前端模块化

  • 公共功能进行封装实现复用
  • 灵活解决依赖关系
  • 解决全局变量污染

如何实现前端模块化

  • js在浏览器端本身是不具备模块化的技术的，需要自己进行封装来实现模块定义以及加载，其实现遵照一定的规范（COMMOMJS）
  • COMMONJS是一个模块化的规范，随着实践分成两种具体的实现Nodejs（CMD），另一种是浏览器端的实现requirejs（AMD）
  • CMD（common module definition）
  • AMD（async module definition）

前端模块化使用requirejs

requirejs：
本身就是几个js文件，它按照AMD规范来实现模块的定义和加载规则

定义模块

通过define()方法定义一个模块

  • 基本使用

  • 依赖注入，[]内是依赖的模块，function里的形参是注入

  • 返回值
    定义模块时可以有返回值，但不是必须的，返回值可以是任何类型的，但是返回对象更有意义

加载模块

通过require()方法加载模块

  • 基本使用
  • 依赖注入

####入口文件
通过为引入require的标签加入data-main属性，其属性值为某一个模块的路径，此模块可以自动被加载并执行

####加载路径
requirejs加载模块时遵照一定的规则，可以分成以下几种情况

  • 当没有入口文件时，加载路径以引入requirejs的页面为准
  • 当有入口文件时，加载路径以入口文件为准
  • 通过配置可以自定义路径
